MultivariateEvents

poissonprocess.time.MultivariateEvents

A class to model a multivariate Poisson process.

Attributes

rates, dict[object, float | int]: a dictionary that maps \(E\), a particular kind of event, to its expected rate of occurrence, \(\mu_{E}\).

Properties

aggregated_rate, float | int: the sum of all the possible events being modelled.

props, dict[object, float]: a dictionary that maps \(E\), a particular kind of event, to its proportion of all events being modelled. Example, if 0.5 was returned for \(E\), then half of all events that occur are expected to be \(E\).

Class methods

from_props (props: dict, aggregated_rate: float | int) -> MultivariateEvents

Convenience method to return an instance of MultivariateEvents using proportions rather than rates.

Precondtions:

  • sum(values(props)) = 1
  • 0 < prop <= 1 for each prop in values(props)
  • aggregated_rate > 0

Methods

describe () -> pandas.DataFrame

Return a descriptive table, where each row shows a kind of event, the event’s rate of occurrence, and the proportion of all events that should be of this type.

n (n_star: int, e: object) -> st.rv_discrete

Return a model for the number of events of the given kind e, given n_star events took place, \(N_{e} \sim B(n^{*}, p_{\text{e}})\).

Preconditions:

  • e in keys(self.rates)

Telephone calls

(M343 Book 2, Activity 5.2)

Let phone calls, MultivariateEvents, be the multivariate Poisson process that models the arrival rate of the different kinds of phone call a tutor receives over the course of an evening.

phone_calls = MultivariateEvents({'student': 2/3,  # per hour
                                  'family': 1/3,
                                  'friend': 1})

Output a descriptive table of the multivariate Poisson process.

phone_calls.describe()
rate prop
kind
student 0.666667 0.333333
family 0.333333 0.166667
friend 1.000000 0.500000
Tip

The class implements the __getitem__ method, which allows us to index into an instance of the class to return models for a particular kind (or kinds) of event.

The returned value from indexing into the instance is an instance of Events.

Return the probability that the two students will call between 7pm and 7.45pm.

phone_calls['student'].x(0.75).pmf(2)
0.075816
Tip

Passing : or ... as an index will return the aggregated model of all kinds of events.

Return the probability that between 7pm and 9pm tomorrow evening, the tutor’s telephone will not ring.

phone_calls[:].x(2).pmf(0)
0.018316

Return the probability that exactly two of the calls are from members of her family, given that the tutor receives four telephone calls one evening.

phone_calls.n(4, 'family').pmf(2)
0.115741

Post office customers

(M343 Book 2, Exercise 5.1)

Let post office, MultivariateEvents, be the multivariate Poisson process that models the arrival of different kinds of customers at the post office over the course of an hour.

Note

We are given the proportions of each kind of event and the global rate, so we use the from_props class method to initialise the instance of MultivariateEvents.

post_office = MultivariateEvents.from_props(props={'letter': 0.7,
                                                   'parcel': 0.05,
                                                   'other': 0.25},
                                            aggregated_rate=8)  # per hour

Output a descriptive table of the multivariate Poisson process.

post_office.describe()
rate prop
kind
letter 5.6 0.70
parcel 0.4 0.05
other 2.0 0.25
Tip

We can index into the rates attribute and props property to return the expected rate and proprotion for a kind of event, respectively.

Return the rate and proportion of all events at which customers arrive at the post office to post parcels.

post_office.rates['parcel'], post_office.props['parcel']
(0.400000, 0.050000)
Tip

Index using a tuple to return a aggregated model for the kinds of events.

Return the probability that over a three-hour period, fewer than five customers arrive to post letters or parcels.

post_office['letter', 'parcel'].x(3).cdf(4)
0.000084

Return the median waiting time between customers arriving to post something (either a letter or a parcel).

post_office['letter', 'parcel'].t.median()
0.115525

Return the probability that the interval between successive customers arriving to post parcels is greater than an hour.

post_office['parcel'].t.sf(1)
0.670320

## Summary

This notebook covers part of M343 Book 2, Part 2, which introduces the Poisson process as a means to model random events that occur in time.

We could not find a Python module that modelled the different type of Poisson processes covered in M343, so we wrote poissonprocesses, a Python processes that handles Poisson processes as summarised by M343.

This notebook covers a **multivariates Poisson process**, which is modelled by the `MultivariateEvents` class.

## MultivariateEvents

> `poissonprocess.time.MultivariateEvents`

A class to model a multivariate Poisson process.

### Attributes

*rates*, `dict[object, float | int]`:
a dictionary that maps $E$, a particular kind of event, to its expected rate of occurrence, $\mu_{E}$.

### Properties

*aggregated_rate*, `float | int`:
the sum of all the possible events being modelled.

*props*, `dict[object, float]`:
a dictionary that maps $E$, a particular kind of event, to its proportion of all events being modelled.
Example, if 0.5 was returned for $E$, then half of all events that occur are expected to be $E$.

### Class methods

> `from_props (props: dict, aggregated_rate: float | int) -> MultivariateEvents`

Convenience method to return an instance of `MultivariateEvents` using proportions rather than rates.

Precondtions:

- *sum*(*values*(*props*)) = 1
- 0 < *prop* <= 1 for each *prop* in *values*(*props*)
- *aggregated_rate* > 0

### Methods

> `describe () -> pandas.DataFrame`

Return a descriptive table, where each row shows a kind of event, the event's rate of occurrence, and the proportion of all events that should be of this type.

> `n (n_star: int, e: object) -> st.rv_discrete`

Return a model for the number of events of the given kind *e*, given *n_star* events took place, $N_{e} \sim B(n^{*}, p_{\text{e}})$.

Preconditions:

- *e* in *keys*(*self.rates*)
